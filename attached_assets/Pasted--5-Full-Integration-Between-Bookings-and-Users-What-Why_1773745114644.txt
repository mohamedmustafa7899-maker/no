#5 - Full Integration Between Bookings and Users

What & Why
The booking flow must be fully connected to authenticated users so that:
- bookings created from the mobile app appear correctly in the admin dashboard
- each logged-in user sees only their own bookings
- each booking is linked to the correct user account
- duplicate client records in db.clients are cleaned up

Current issues:
- BookingScreen does not attach bookings to the logged-in user
- GET /api/bookings returns all bookings to every user
- POST /api/bookings does not save userId correctly
- POST /api/bookings creates/updates client records incorrectly using nextId instead of the logged-in user's real id
- db.clients contains duplicate records (including duplicate id=1)

Done looks like
- When a user books from the app, the booking appears immediately in the dashboard under Bookings
- “My Bookings” shows only the current logged-in user’s bookings
- If the user is logged in, their name is auto-filled in the booking form
- userId is saved on every booking
- db.clients is cleaned so duplicate records are removed
- No more client/user ID mismatch for logged-in users

Out of scope
- Push notifications
- Payments for bookings
- Rebuilding the booking system from scratch
